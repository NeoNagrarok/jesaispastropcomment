1.initialisation du projet:
 
	Le projet est initialisé suite a une réunion, l’admin crée le dépôt ainsi que les différentes branches (une branchepar développeur)
	Le projet est subdivisé par fonctionnalité (module/ticket) et attribués aux développeurs.
		trello kanban autre
 
 2.définition des rôles:	 
	 a. admin:
		crée le dépôt
		Faire les invites à partir d’une liste formatée
		gère les branches / le dev / la prod ???
			valide les pull request et les push sur le master
	 
	 b. responsable qualité:
	 validation du fonctionnement en dev avant validation du push en production
 
D. développeur
	Process
		Pour chaque ticket/module dans trello (kanban)
		Création d’une branche avec charte de nomage
		Developpement la branche respective
		Push et soumission en qualitéd. Application
		Github / GitKrakken / autres
		VsCode
 
 
 
3.déroulement du projet:

Chaque développeur code sa fonctionnalité demander et push sur sa branches
il peuvent crée des sous-branche pour sous-diviser le projet.Quand ils ont fini leur taches , ils font un pull request qui sera alors étudié
par le responsable qualité et/ou l’administrateur qui le validera et l’intègrera à la branche master une le contrôle effectuer fois fini.
